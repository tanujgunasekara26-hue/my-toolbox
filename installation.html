⚙️ Installation Guide

Get My Toolbox up and running in minutes

System Requirements

OS Windows 10 or later
CPU Intel Core i5 (2nd Gen) or better
RAM 8GB minimum
Storage 500MB free space
Internet Optional (required for AI & Currency tools)
Architecture 64-bit recommended

Installation Method 1: Using the Installer (Recommended)

This is the easiest way to get started. No technical knowledge required.

Steps

  1. Download the latest MyToolbox-Setup.exe from the Download page or GitHub Releases
  2. Run the downloaded .exe file
  3. If Windows SmartScreen appears, click "More info" → "Run anyway" (this is a known false positive for fresh builds)
  4. Follow the on-screen installer prompts (accept license, choose install location, etc.)
  5. Wait for the installation to complete
  6. Launch My Toolbox from the Start Menu or Desktop shortcut
✅ You're done! The app is ready to use. Pick a theme and start exploring all 21 tools.

Installation Method 2: Running from Source

For developers, contributors, or anyone who prefers running from source code.

Prerequisites

  • Python 3.14 or later (download here)
  • Git (to clone the repo; download here)
  • Basic command-line knowledge

Steps

  1. Clone the repository:
    git clone https://github.com/tanujgunasekara26-hue/my-toolbox.git
  2. Navigate to the project directory:
    cd my-toolbox
  3. Install dependencies:
    pip install -r requirements.txt --break-system-packages
  4. Run the application:
    python main.py
💡 If you get a ModuleNotFoundError, make sure you're in the correct directory and all dependencies installed successfully. Re-run step 3 if needed.

Installation Method 3: Using a Virtual Environment (Python)

Recommended if you want to isolate My Toolbox dependencies from your system Python.

Steps

  1. Clone and navigate to the repo (same as Method 2, steps 1-2)
  2. Create a virtual environment:
    python -m venv venv
  3. Activate the virtual environment:
    venv\Scripts\activate
  4. Install dependencies (no --break-system-packages needed here):
    pip install -r requirements.txt
  5. Run the app:
    python main.py
💡 To deactivate the virtual environment later, just run: deactivate

Windows SmartScreen Warning

Fresh builds of My Toolbox may trigger Windows SmartScreen. This is a false positive — the app is safe.

How to Bypass It

  1. When the SmartScreen popup appears, click "More info"
  2. Click "Run anyway" at the bottom right
  3. Installation will proceed normally
⚠️ If you don't see the "Run anyway" button, try downloading directly from our GitHub releases page rather than other sources.

Troubleshooting

❌ "ModuleNotFoundError: No module named 'PySide6'"

Solution: Dependencies didn't install. Re-run:

pip install -r requirements.txt --break-system-packages

❌ App Won't Start / Crashes Immediately

Solution: Make sure you meet the system requirements above. If you do, try:

  • Uninstall and reinstall using the latest version from Downloads
  • Delete the data/ folder (stores preferences/notes) and restart
  • Check the console for error messages (run from source and look for Python tracebacks)

❌ "Permission Denied" or "Access Denied" on Windows

Solution: Try running as Administrator:

  • Right-click the shortcut or .exe file
  • Select "Run as administrator"

❌ AI Features Not Working

Solution: See the AI Features page for setup instructions. You need:

  • A Cerebras API key (sign up at cloud.cerebras.ai)
  • The key added as an environment variable: CEREBRAS_API_KEY
  • To be in the United States

❌ Still Having Issues?

Report the issue on GitHub with:

  • Your Windows version
  • Python version (if running from source)
  • Full error message or console output
  • What you were trying to do

Uninstalling My Toolbox

If Installed via the Installer

  1. Press Win and search for "Control Panel"
  2. Go to Programs → Programs and Features
  3. Find "My Toolbox" in the list
  4. Click it and select "Uninstall"
  5. Confirm and wait for uninstall to complete
💡 Your notes and preferences are stored in %APPDATA%\MyToolbox\data\. They won't be deleted during uninstall if you want to keep them.

If Running from Source

Simply delete the folder containing the cloned repository. No cleanup needed.
