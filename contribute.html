🚀 Getting Started

  1. Fork the repository on GitHub
  2. Clone your fork locally
  3. Create a new branch for your change
  4. Make your changes and test them locally
  5. Commit, push, and open a pull request
git clone https://github.com/tanujgunasekara26-hue/my-toolbox.git
cd my-toolbox
git checkout -b feature/your-feature-name
pip install -r requirements.txt
python main.py
💡 Tip: Open an issue first to discuss larger changes before investing significant time — it helps avoid duplicated effort.

📋 Contribution Guidelines

  • Keep pull requests focused on a single change or fix
  • Follow the existing code style used in each tool module
  • Test your changes across the affected tool(s) before submitting
  • Write clear, descriptive commit messages
  • Update documentation if your change affects usage or behavior
  • Be respectful and constructive in discussions and code reviews
⚠️ Contributions that include or rely on additional Microsoft-owned assets must respect the same non-commercial restrictions described in the EULA.
